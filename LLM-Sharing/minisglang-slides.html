System Architecture

User RequestAPI / HTTP
Scheduler (Control Plane)Tokenization, Batching, Prefix Match
Engine (Data Plane)Forward Pass, KV Cache, Weights
GPU (CUDA)Paged Attention, CUDA Graphs
  • Scheduler: Decides "what" to run. Manages the high-level logic and request queue.
  • Engine: Handles the "how." Owns GPU memory and actual model execution.
  • Separation: Decoupling management from execution to maximize GPU occupancy.
  • Async Flow: Frontend prepares next batch while Backend processes current one.

The Magic of RadixTree

Re-computing KV caches for shared prompts is wasteful. mini-sglang solves this with a RadixTree-based manager.

  • Automatic O(match_len) matching.
  • LRU-based cache eviction policy.
